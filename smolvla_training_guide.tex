\section*{Overview}
All workflow scripts live in \texttt{scripts/} and are numbered in execution order.

\section{Script Order}
\begin{enumerate}
    \item \texttt{scripts/00\_run\_params.sh} (edit only)
    \item \texttt{scripts/01\_sync\_to\_gpu.sh} (run locally)
    \item \texttt{scripts/02\_preflight\_gpu.sh} (run locally, recommended)
    \item \texttt{scripts/03\_setup\_gpu.sh} (run on GPU node)
    \item \texttt{scripts/04\_start\_training.sh} (run on GPU node)
    \item \texttt{scripts/05\_extract\_from\_scratch.sh} (run locally after training)
\end{enumerate}

\section{1) Edit Parameters}
Edit the committed template \texttt{scripts/00\_run\_params.sh}, then create a local override:
\begin{lstlisting}[language=bash]
cp scripts/00_run_params.local.example.sh scripts/00_run_params.local.sh
\end{lstlisting}

Use \texttt{scripts/00\_run\_params.local.sh} for personal values (gitignored).

Key fields:
\begin{lstlisting}[language=bash]
WORKFLOW_USER="your_ucl_username"
REMOTE_PROJECT_DIRNAME="smolvla_project"
RUN_NAME="001_002_003"
DATASET_NAMES=(001 002 003)
DATASET_ROOT="/scratch0/${WORKFLOW_USER}/lerobot_datasets"
SAVE_FREQ=1000
EXTRACT_FOLDER_NAME="${RUN_NAME}_smolvla_full"
\end{lstlisting}

\section{2) Local Steps}
\begin{lstlisting}[language=bash]
bash scripts/01_sync_to_gpu.sh
bash scripts/02_preflight_gpu.sh
\end{lstlisting}

Helper block:
\begin{lstlisting}[language=bash]
cp scripts/00_run_params.local.example.sh scripts/00_run_params.local.sh
$EDITOR scripts/00_run_params.local.sh
bash scripts/01_sync_to_gpu.sh
bash scripts/02_preflight_gpu.sh
\end{lstlisting}

\section{3) Connect to GPU}
\begin{lstlisting}[language=bash]
source scripts/00_run_params.sh
ssh -l "$REMOTE_USER" -J "$REMOTE_USER@$JUMP_HOST" "$GPU_NODE"
\end{lstlisting}

\section{4) Remote Steps (on GPU)}
\begin{lstlisting}[language=bash]
cd ~/smolvla_project/SmolVLA-Testing/scripts
bash ./03_setup_gpu.sh
bash ./04_start_training.sh
\end{lstlisting}

Helper block:
\begin{lstlisting}[language=bash]
cd ~/smolvla_project/SmolVLA-Testing/scripts
bash ./03_setup_gpu.sh
bash ./04_start_training.sh
\end{lstlisting}

Resume mode:
\begin{lstlisting}[language=bash]
bash ./04_start_training.sh --resume
\end{lstlisting}

\section{5) Extract After Training (Local)}
\begin{lstlisting}[language=bash]
bash scripts/05_extract_from_scratch.sh
\end{lstlisting}

The extraction script aborts if destination exists and is non-empty.

\section{Platform Compatibility}
\begin{itemize}
    \item macOS: supported.
    \item Linux: supported.
    \item Windows: supported via WSL or Git Bash/MSYS2 with \texttt{ssh} and \texttt{rsync} installed.
    \item Plain PowerShell/CMD without a Bash+rsync environment is not supported by these scripts.
    \item In WSL, keep the repository inside the Linux filesystem for better performance.
\end{itemize}

\section{Known Constraints}
\begin{enumerate}
    \item One literal \texttt{rsync} invocation cannot send code to home and data to scratch simultaneously.
    \item \texttt{scripts/01\_sync\_to\_gpu.sh} provides one-command UX by doing two internal \texttt{rsync} calls.
    \item SSH prompts depend on key/password auth configuration.
\end{enumerate}
